## 所需工具

- [BG3 模组作者多功能工具](https://github.com/ShinyHobo/BG3-Modders-Multitool)
- [BG3 迷你工具](https://www.nexusmods.com/baldursgate3/mods/502)
- [VSCode](https://code.visualstudio.com/) 或 [Notepad++](https://notepad-plus-plus.org/)
- VSCode 中的 UUID 生成器扩展，或者 Notepad++ 中的 ToolBucket 和 XML Tools（通过 插件 > 插件管理 搜索并安装）。

要在 VSCode 中生成 UUID，您可以高亮显示某个部分或在空白区域单击，右键单击，然后选择 在此处生成 UUID。

对于 Notepad++ 中的 UUID 生成，安装 ToolBucket 后，按 Alt + Shift + G。它将弹出一个窗口，如下所示。保持 **Include hyphens**（包含连字符）勾选，如果您不使用其他选项，也可以单击 **Don't ask again**（不再询问）。然后单击 生成。

或者使用在线 [UUID-生成器](https://generateuuid.net/#uuidv4)。

注意：Larian 似乎使用 RFC 4122 变体 2 版本 4 UUID 的组合，以及完全随机的 128 位序列。

## Meta.lsx

大多数 pak 模组都需要一个 meta.lsx。meta.lsx 是一个文件，用于告诉模组加载器有关您的模组的某些“元”信息。这些信息包括作者姓名、版本号等。

文件路径应为：_ModName_ /Mods/_YourShared_ /meta.lsx

_ModName_ 文件夹是您模组的工作空间文件夹。每个模组都需要一个，它包含所有其他文件夹。然后您需要创建一个名为 Mods 的文件夹，再创建一个自定义名称的新文件夹（可以与模组名称相同），这将被称为 _YourShared_ 文件夹，最后您需要创建一个 .txt 文件并将其重命名为 meta.lsx。

如果您的模组被归类为基础游戏覆盖，则唯一不需要 meta.lsx 的情况。您可以通过 BG3 模组管理器的“覆盖”部分中放置的模组或具有柑橘色高亮的模组来识别它们。在某些情况下，您可能仍然添加 meta，以便人们可以将其放入他们的加载顺序中，如果他们愿意的话。

### 手动创建您的 meta.lsx

以下是 meta.lsx 的模板。只需将其粘贴到您之前创建的 meta.lsx 文件中，然后按照说明编辑模板。

```xml
<?xml version="1.0" encoding="UTF-8"?>

<save>
<version major="4" minor="0" revision="9" build="331"/>
<region id="Config">
<node id="root">
<children>
<node id="Dependencies"/>
<node id="ModuleInfo">
<attribute id="Author" type="LSString" value="AUTHOR NAME HERE"/>
<attribute id="CharacterCreationLevelName" type="FixedString" value=""/>
<attribute id="Description" type="LSString" value="DESCRIPTION HERE"/>
<attribute id="Folder" type="LSString" value="MOD FOLDER NAME HERE"/>
<attribute id="LobbyLevelName" type="FixedString" value=""/>
<attribute id="MD5" type="LSString" value=""/>
<attribute id="MainMenuBackgroundVideo" type="FixedString" value=""/>
<attribute id="MenuLevelName" type="FixedString" value=""/>
<attribute id="Name" type="LSString" value="MOD NAME HERE"/>
<attribute id="NumPlayers" type="uint8" value="4"/>
<attribute id="PhotoBooth" type="FixedString" value=""/>
<attribute id="StartupLevelName" type="FixedString" value=""/>
<attribute id="Tags" type="LSString" value=""/>
<attribute id="Type" type="FixedString" value="Add-on"/>
<attribute id="UUID" type="FixedString" value="UUID HERE"/>
<attribute id="Version64" type="int64" value="36028797018963968"/>
<children>
<node id="PublishVersion">
<attribute id="Version64" type="int64" value="36028797018963968"/>
</node>
<node id="TargetModes">
<children>
<node id="Target">
<attribute id="Object" type="FixedString" value="Story"/>
</node>
</children>
</node>
</children>
</node>
</children>
</node>
</region>
</save>
```

让我们回顾一下您需要编辑的行：

<node id="Dependencies"/>

请通读 [依赖项](Modding_colon_Dependencies.md "模组：依赖项") 了解此部分以及它是否与您的模组相关。

<attribute id="Author" type="LSString" value="AUTHOR NAME HERE"/>

在此行中，您需要输入您的作者姓名。这将显示在模组管理器的“作者”列中。可以是任何内容，但大多数人使用他们的 nexusmods 用户名。

<attribute id="Description" type="LSString" value="DESCRIPTION HERE"/>

这是当用户将鼠标悬停在 BG3 模组管理器中的模组上时显示的简短描述。

<attribute id="Folder" type="LSString" value="MOD FOLDER NAME HERE"/>

这需要与我们之前提到的 _YourShared_ 文件夹的名称匹配。如果您输入不正确，模组中的内容将损坏。

<attribute id="Name" type="LSString" value="MOD NAME HERE"/>

此处输入您的模组名称。不要在 b 前面加 -，因为我们最近发现这会导致模组无法加载。

<attribute id="UUID" type="FixedString" value="UUID HERE"/>

为此行生成一个唯一的 UUID。这是模组管理器区分模组的方式，因此此 UUID 必须是唯一的，除非您希望它覆盖另一个模组。

<attribute id="Version64" type="int64" value="36028797018963968"/>

meta.lsx 中的版本号采用 Int64 格式。当由 BG3 模组管理器读取时，此行读取为 1.0.0.0。您可以使用 BG3 模组管理器、多功能工具或 Padme 的迷你工具生成此 Int64 版本号。这也是您每次更新时需要更改的行。

<node id="PublishVersion">
<attribute id="Version64" type="int64" value="36028797018963968"/>
</node>

此版本号是发布时的版本。即使更新后也可以保留此数字。但是，您可以使其与更新号相同。此功能的存在更多是为了 Larian 而不是我们。这是因为如果他们在开发过程中制作新的 pak，他们的 `PublishVersion` 将是创建该 pak 时的当前游戏版本。

如果您的模组依赖于另一个模组的资产（请记住获得许可），最好将该模组添加为依赖项，请在此处了解[方法。](Modding_colon_Dependencies.md "模组：依赖项")

您可以使用多功能工具、BG3 模组管理器或 Padme 的迷你工具生成版本号。方法如下：

#### 多功能工具

打开后，转到 工具 > 版本计算器，如下所示：

打开后您将看到此界面：

使用 主版本/次版本/修订版/构建号 的上下箭头，它将更改下面的数字。设置为您想要的版本后，使用第二行的复制按钮并将其粘贴到您的 meta 中的此行的第一部分：

#### BG3 模组管理器

在 BG3 模组管理器中，转到 工具 > 切换版本生成器窗口，或者在打开时按 Ctrl + G 打开它。如下所示：

打开后您将看到此界面：

使用 主版本/次版本/修订版/构建号 的上下箭头，它将更改下面的数字。设置为您想要的版本后，使用复制按钮并将其粘贴到您的 meta 中的此行的第一部分：

<attribute id="Version64" type="int64" value="36028797018963968"/>

#### BG3 迷你工具

转到 额外工具 按钮

它将打开迷你工具的当前额外工具。

使用第一个和最后一个数字的上下箭头，它将更改下面的数字。设置为您想要的版本后，要么高亮显示框中的文本并复制粘贴到您的 meta 中的此行的第一部分：

<attribute id="Version64" type="int64" value="36028797018963968"/>

或者使用 **保存到** 按钮，然后找到您的 meta.lsx，它将为您覆盖这些行。

### 使用多功能工具创建您的 meta.lsx

您的模组完成并准备好在游戏中测试后，在您的工作空间文件夹内创建一个 Mods 文件夹，然后将您的工作空间文件夹拖到蓝色框上，该框显示“将模组工作空间文件夹或 mod.pak 放置在此处”。如果您的 Mods 文件夹没有 meta.lsx，它将弹出此弹出窗口：

在“作者”框中，输入您在 nexusmods 上使用的名称，或您想要的任何名称。

在“描述”中，简要描述您的模组，可以是任何内容。

然后选择您的模组的版本号。准备就绪后，您可以单击确认按钮。

如果想要更新版本号，您可以删除您的 meta.lsx 并再次将您的工作空间文件夹拖到蓝色框上，这将再次触发此弹出窗口。

### 使用 Padme4000 的 bg3 迷您工具创建您的 meta.lsx

如果您觉得跟随视频更容易，或者希望视频与书面教程一起使用，请在此处找到我的视频[链接](https://youtu.be/tanDAq3JKPU?si=P4aJ3Rul4MeKIxdU)，了解如何使用该工具的该部分。

打开工具后，您将看到一个名为 meta 的按钮，这就是我们想要的。

如果您还没有设置文件夹，可以使用此窗口的顶部创建文件夹。它们将在工具 exe 所在的 Mods 文件夹中创建。如果该文件夹尚不存在，它将创建该文件夹。

模组创建者

- 输入您想要与模组关联的名称，例如您的 nexusmods 名称。输入完成后单击更新。

始终单击更新。

模组名称

- 行为您的模组提供一个名称，该名称将显示在模组管理器中。

模组描述

- 可以是任何内容，只需简要描述或不添加任何内容。

共享文件夹 - 您希望此行与您为辅助文件夹指定的名称相同。

  - 我们将使用 meta 路径作为示例：
- ModName/Mods/YourShared 最后一个文件夹需要与您给该行的名称匹配

唯一 UUID

  - 使用生成的唯一 UUID 按钮，然后在生成后单击更新。
- 您希望它是唯一的，以免与其他模组冲突

版本号

- 默认设置为 1.0.0.0，以防文件有任何更改，请单击更新按钮将其设置回 1.0.0.0

现在您可以单击另存为，然后将文件保存到 ModName/Mods/YourShared 文件夹中。

恭喜！您的 meta.lsx 已准备就绪。
