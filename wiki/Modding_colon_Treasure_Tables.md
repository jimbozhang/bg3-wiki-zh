## 宝藏表

对模组制作者工具包至关重要的众多文本文件类型之一是宝藏表。宝藏表定义了游戏中的所有库存——其中包含什么以及可能包含什么。所有箱子、商人和尸体都使用宝藏表来定义其战利品。对于添加物品的模组，其在游戏中出现的主要方式是将它们添加到一个或另一个宝藏表中。最常见的情况是，仅为了添加物品而添加物品的模组会将其物品添加到教程箱中，这是一个在鹦鹉螺上找到的软骨箱。本页将介绍宝藏表的基础知识。

### 示例 1

让我们看看游戏中一个 TreasureTable.txt 文件的摘录：

```
new treasuretable "Gold_Pocket_Wealthy"
new subtable "10,1;11,1;12,1;13,1"
object category "Gold",1,0,0,0,0,0,0,0
```

第一行，以一种相当自解释的方式，创建了一个名为 "Gold_Pocket_Wealthy" 的新宝藏表。

第二行看起来更令人困惑，但其实不然。表中的每个项目都需要一个子表。如果你想在宝藏表中保证有一个治疗药水，它仍然需要自己的子表。因此，考虑到这一点，创建了一个子表。引号内的数字表示表将被抽取的次数的权重——例如，此表的权重以 "10,1" 开始，因此，权重为 1（等于表中的其他所有内容），此表可以被抽取 10 次。这意味着我们将从表中列出的任何内容中获得 10 个东西。每个后续的编号对表示另一个数量和权重对——"11,1" 表示权重为 1，表将被抽取 11 次，依此类推。

最后一行，与第一行类似，非常自解释。它以 "object category" 开头，这对于子表中的所有项目都是必需的——对我们来说意义不大，因为它总是保持不变，但对代码来说意义重大。"Gold" 很明显指的是游戏中的货币黄金。1 是该项目在表内的权重——因此，如果有多个项目，每个项目被选中的概率就是这些权重。如果有黄金，权重为 1，以及治疗药水，权重为 2，那么每次抽取表时，黄金被选中的概率是 1/3，而治疗药水被选中的概率是 2/3。因此，如果在抽取表时得到 13，那么子表将被抽取 13 次，每次抽取有 2/3 的概率获得治疗药水，1/3 的概率获得黄金。

### 示例 2

让我们再看一个展示几个不同概念的例子：

```
new treasuretable "Supply_CampFood"
new subtable "0,1;1,1;2,1;3,1"
object category "I_CONS_FOOD_Vegetable_Potato",1,0,0,0,0,0,0,0
object category "I_CONS_FOOD_Vegetable_Carrot",1,0,0,0,0,0,0,0
new subtable "1,1"
object category "T_ST_Food_Sausage",1,0,0,0,0,0,0,0
```

这个表有点长，但由于我们已经理解了一些概念，我们不需要全部讲解。另外请注意，这只是游戏代码中表的一部分，用于展示重要概念。

前两行创建了一个名为 "Supply_CampFood" 的新宝藏表，有 1/4 的概率获得 0、1、2 或 3 个下面列出的东西。第一个需要注意的重要事项是项目名称前的 "I\_"。这不是游戏中项目名称的一部分，而是表示它是一个项目。

因此，在前 4 行中，我们的宝藏表现在有 0、1、2 或 3 个胡萝卜或土豆。下一行展示了另一个（相当明显的）但重要的概念——他们希望某物保证被抽取恰好一次，因此我们现在知道下一行保证在表中。

因此，第 6 行也是最后一行，看起来很像第 3 行和第 4 行，但有一个小区别——之前的 "I\_" 被替换为 "T\_"。这个 "T\_" 表示该行调用了另一个先前创建的表。因此，在这种情况下，该行调用了表 "ST_Food_Sausage"。搜索文本文件的其余部分，你可以找到该表如下所示：

```
new treasuretable "ST_Food_Sausage"
new subtable "1,1"
object category "I_CONS_FOOD_Sausage",2,0,0,0,0,0,0,0
object category "I_CONS_FOOD_Sausage_B",2,0,0,0,0,0,0,0
object category "I_WPN_Salami",3,0,0,0,0,0,0,0
object category "I_CONS_FOOD_Sausage_D",2,0,0,0,0,0,0,0
```

因此，此表有 2/9 的概率产生 "Sausage"，2/9 的概率产生 "Sausage_B"，1/3 的概率产生 "Salami"，以及 2/9 的概率产生 "Sausage_D"。

有了所有这些信息，我们可以说我们原始的表 "Supply_CampFood" 不仅包含我们之前找到的 0、1、2 或 3 个土豆和胡萝卜，还包含 1 个任何 4 种香肠类型之一，权重如上所列。
