## 示例模板

请记住，最佳实践是解包并直接查阅游戏文件。

- [Mod Builder](https://www.nexusmods.com/baldursgate3/mods/3862) 由 ReallyLazyIcarus 开发。这是一个 Python 脚本，可生成文件夹工作区和空的 lsx 文件，格式正确，可使用多工具转换。推荐！
- [BG3 Equipment Generator](https://www.nexusmods.com/baldursgate3/mods/5165) 由 Bytebard 开发。这是一个新程序，看起来是添加物品的一种有前途的新方法。
-

不幸的是，许多模板护甲模组已经过时，通常是因为模组工具变得更加高效。例如，这些模组中提到的大多数[种族 UUID](https://bg3.wiki/wiki/Modding:Race_UUID) 已失效，有些元文件（meta）无法工作。它们仍然可用于学习，大部分代码仍然有效，但不建议将它们用作模板，仅作为学习工具。例如，示例装备模组（Sample Equipment Mod）的背包模板仍然可以正常工作。

- [Bububull 的懒人护甲模组模板](https://www.nexusmods.com/baldursgate3/mods/10554) 由 Bububull 开发。
- 一个包含许多示例服装物品以及如何将它们放入教程宝箱的新模板。

  - 一个用于数据挖掘和 3D 编辑护甲模组的模板模组。包含 1 件护甲、1 双不匹配的靴子、1 双不匹配的手套、1 件头饰和 1 件肩甲。主要作为模组制作者资源，但玩家也可以享受。
- [示例魔法戒指模组](https://www.nexusmods.com/baldursgate3/mods/278) 由 AnteMaxx 开发
  - 一个用于魔法戒指和法术模组的模板模组。包含 4 个魔法戒指。主要作为模组制作者资源，但玩家也可以享受。
- [示例装备模组](https://www.nexusmods.com/baldursgate3/mods/249) 由 AnteMaxx 开发
  - 一个用于护甲和武器模组以及实现自定义图标的模板模组。包含 3 件护甲、1 件武器和 1 个背包。主要作为模组制作者资源，但玩家也可以享受。
- [BG3 Creator's Cauldron](https://www.nexusmods.com/baldursgate3/mods/1808)- 一个较旧的程序。因其包含护甲.txt 和被动的长列表而有用，供你使用。
- [BG3 MiniTool by Padme4000](https://www.nexusmods.com/baldursgate3/mods/502)- 可以生成一个[元文件](Modding_colon_Creating_meta.lsx.md "Modding:Creating meta.lsx")供你使用。

## Meshes.lsx

首先，我们从 Blender 导出我们的模型 **HUM_M_CLT_Headwear_Circlet_Silver_A** 到 MySweetMod/Generated/Public/MySweetMod/**Assets** 文件夹。

我们现在为 MySweetMod\\Public\\MySweetMod\\Content\\Assets\\Characters\*\*[PAK]\_Armor\*\* 创建 2 个 lsf.lsx。

这些将包含一个 visualbank（模型列表）、一个 materialbank（材质列表）和一个 texturebank（纹理列表）。你可以将它们拆分成单独的页面或全部组合在一起，这无关紧要。当多工具转换它们时，它们都会被合并在一起。这种分离是为了方便查看，因为它们可能会变得很长。为了满足我们的需求，我们将它们放入 2 个 lsx 中，一个用于模型，一个用于纹理/材质。

现在我们找到的材质，SourceFile "Public/Shared/Assets/Materials/Characters/CHAR_BASE.lsf" 是一个相当标准的着色器，没有透明度或发光。可以透明的材质标题中带有 'Alpha'。例如，"Public/Shared/Assets/Materials/Characters/CHAR_BASE_AlphaTest_2S.lsf" 或 "CHAR_BASE_AlphaTest_2S_Dither" 将启用 alpha 并使模型从两侧可见（关闭背面剔除）。内衣需要使用 Dither 着色器，否则无法正确隐藏。

以 \_VT 结尾的材质用处较小，因为它们仅适用于虚拟纹理。虚拟纹理本质上是一个纹理容器，可以更快地加载。我们可以使用 Script Extender 和 Lslib 打包我们自己的虚拟纹理，但这超出了本页的范围。

为了使裤子在穿上靴子时消失，你可能需要使用 \_Vertcut 材质，例如 'Public/Shared/Assets/Materials/Characters/CHAR_BASE_VertCut.lsf'，并确保在模型 lsx 中将 VertexMask 设置为 True。（以及将物品本身顶点绘制为所需颜色）

## 护甲和被动

**MySweetMod\\Public\\MySweetMod\\Stats\\Generated\\Data\\Armor.txt**

**MySweetMod\\Public\\MySweetMod\\Stats\\Generated\\Data\\Passive.txt**

同时在你的 **Armor.txt** 中，它们像这样链接：

你可以查看游戏自带的 armor.txt 以获取此处应填写内容的思路。你可以删除不想添加的内容，你真正需要的只是条目、类型、使用的插槽和 RootTemplate。

Armor.txt
new entry "MY_COOL_NEW_CIRCLET"
type "Armor" **注意 'type' 部分可以使物品在游戏随机放置物品的地方生成。如果你希望它永远不会在其他地方生成，可以将 type 设置为 ""**。或者，可能在下面添加 data "ItemGroup" ""。
data "ValueOverride" "4000" **(这是一个非常酷的饰环，我们希望它价格很高！)**
data "Weight" "0.01"
using "\_Head_Magic_Circlet" **(这告诉它在哪个插槽装备。理想情况下，这应该与你设置的 ParentTemplate 匹配。)**
data "RootTemplate" "0000000000000000000000000002" **与 rootemplate Mapkey 匹配。**
data "Rarity" "Rare" **这影响它在你物品栏中的颜色，稀有物品会有不同的颜色轮廓。**
data "Boosts" "UnlockSpell(Target_MAG_HealingWord);UnlockSpell(Shout_MAG_HealingWord_Mass)" **这强制它给我们这些法术。注意角色需要法术位才能使用它们！如果你希望它适用于所有人，最好将其设置为自定义被动**。
data "PassivesOnEquip" "MAG_Radiant_Radiating_Helmet_Passive;CUSTOMPASSIVEIMADE" **( ; 分隔被动。)**
data "Unique" "1" **(如果你在多个地方放置它，这些可能会让你头疼，因为它们不会为忽略第一个的人生成。因此我不建议使用这些！)**
data "MinAmount" "1"
data "MaxAmount" "1"

其他 `using` 插槽包括 `"ARM_Camp_Body"`、`"_Vanity_Shoes_Generic_Common"` 用于营地服装、`"_Foot_Magic"` 用于靴子、`"ARM_Robe_Body"` 用于长袍、`"_Hand"` 用于手套。

如果添加自定义被动，你的 **Passive.txt** 将如下所示。你根本不需要 passive.txt，你可以通过重用游戏的被动来制作完美的物品，这只是为有抱负的人准备的。

示例被动代码借用自 Isobel 的护甲。

new entry "CUSTOMPASSIVEIMADE"
type "PassiveData"
data "DisplayName" "HANDLE3;1" **(在你的 .loca 文件中为此生成一个句柄。)**
data "Description" "HANDLE4;1" **在你的 .loca 文件中为此生成一个句柄。)**
data "Icon" "CUSTOMPASSIVEICON" **(我们可以像设置物品图标一样设置它)**
data "DescriptionParams" "DealDamage(1d4, Radiant)"
data "StatsFunctorContext" "OnAttacked"
data "StatsFunctors" "IF(context.HasContextFlag(StatsFunctorContext.OnAttacked) and HasStatus('MAGE_ARMOR') and IsLastConditionRollSuccess(ConditionRollType.ConditionSavingThrow) and not Self() and IsSavingThrow()):DealDamage(SWAP, 1d4, Radiant,Magical)"

## Loca

**MySweetMod\\Localization\\English\\MySweetMod.loca.xml**

你可以通过勾选多工具上 'GENERATE' 旁边的 Handle 框来生成句柄。它很像 UUID，但以 H 开头。版本号与内部引用有关。只需让你的所有句柄都说 1。

</contentList>

## TreasureTable

**MySweetMod\\Public\\MySweetMod\\Stats\\Generated\\TreasureTable.txt**

现在，你需要将它放入 treasuretable。

你的 treasuretable 应该像这样：

new treasuretable "TUT_Chest_Potions"
CanMerge 1
new subtable "1,1"
object category "I_MY_COOL_NEW_CIRCLET",1,0,0,0,0,0,0,0

这将使它在教程宝箱中生成。其他将物品放入不同位置的 treasuretable 代码可以在这里找到[- 模组制作者资源 - Treasure Table 电子表格](https://www.nexusmods.com/baldursgate3/mods/2534)。

确保不要忘记物品的 _I_\_ 前缀！

数字影响掉落率等。上面的数字将导致物品始终在宝箱中生成。更多信息请参见 Larian 关于[TreasureTables](https://docs.larian.game/Treasure_Tables)的维基页面，它适用于不同的游戏，但理念相同。

如果你计划在你的模组中放置大量物品，建议创建一个包含它们的背包物品，并将其放入教程宝箱，以避免杂乱。

如果出现问题，建议覆盖其他人的自定义模组，或者将其添加到现有模组的末尾。当你知道问题不在于你的模型时，检查你的材质，这样你就可以一步一步地修复问题。
