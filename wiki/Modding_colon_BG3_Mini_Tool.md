## 工具功能及其使用方法

### 使用迷你工具创建/编辑 .loca

如果您不想手动编辑 .loca，可以使用 Padme 的迷你工具来完成！通过 CharacterCreationAppearanceVisuals 按钮打开本地化编辑器，并在其中找到打开本地化按钮。

打开后如下所示：

首先我们看到文件夹路径设置，如果您还没有，请为 .loca 文件创建文件夹。不过在我们编辑时，扩展名是 .xml。

如果您想使用工具创建文件夹，请在工具 exe 所在的同一文件夹中创建一个名为 Mods 的文件夹。并将您的模组文件夹放在这里。现在对于编辑器中的第一个框，填写您的工作区文件夹的确切名称，然后您可以检查模组文件夹以查看它是否能看到它们，如果您没有它们，请单击创建文件夹。该工具将创建缺少的文件夹。

对于名称句柄，您首先要单击生成唯一句柄，一旦它有一个，就单击更新按钮。请记住始终单击更新按钮。并且完成后始终检查您的文件，以防工具因任何原因无法工作或您忘记按更新，因为您不希望 UUID/句柄冲突。

对于显示名称，您需要写入当您的资源处于活动状态时希望用户阅读的内容。例如在角色创建中，它是当您将鼠标悬停在资源上时显示的名称。完成后，单击更新按钮。

如果您还没有本地化文件，请使用另存为按钮。

如果您已经设置好了一个，首先使用定位功能定位您的 localization.xml，上面的框将填充该路径。然后单击添加到您的模组文件将新句柄添加到您的模组。

恭喜！您已完成设置/使用迷你工具来编辑/创建您的本地化文件。

请记住在打包之前使用 lslib 将 xml 转换为 .loca，或者如果使用 multitool 打包，请将文件扩展名设为 .loca.xml，以便 multitool 自动为您转换。

### 使用迷你工具创建您的 CharacterCreationAppearanceVisuals

如果您使用 Padme 的迷你工具，可以通过打开工具并转到 CharacterCreationAppearanceVisuals 按钮来自动创建这些插槽。

打开后如下所示：

首先，如果您还没有创建文件夹，请创建文件夹。如果还没有，此工具允许您创建文件夹，它将在工具目录中创建一个名为 Mods 的文件夹。

如果您在任何时候遇到困惑，请随时单击 ? 按钮以获取更多信息。但准备好文件夹后，您可以选择您的资源正在为其制作的体型。

- 默认 = 种族使用的默认体型。在这种情况下，龙裔和半兽人使用默认体型而非其他体型。
- 其他 = 他们使用的其他体型，例如强壮体型

体型类型，您希望资源用于男性还是女性体型。

接下来是句柄。单击生成唯一句柄或复制您在本地化文件中创建的句柄。或者，如果您还没有设置本地化文件，可以使用打开本地化编辑器。准备好后，单击更新按钮。

完成一个部分后始终按更新。

RaceUUID - 这是定义资源所用种族的 ID。单击下拉菜单获取如下选项：

默认情况下，此列表仅包含原版种族。最后一个选项是用于将自定义种族 UUID 添加到列表的占位符示例，以便您可以从工具的此部分使用它。如果您按照下一个标题部分添加任何内容，单击列表时它将更新。有时可能需要双击。

从下拉菜单中选择您要使用的种族，然后单击更新。

我们的下一个下拉菜单用于插槽名称：

您可能需要单击两次才能显示整个列表。对于我们的需求，我们希望选择头发。然后单击更新。但列表的其余部分是可以通过 CCAV 添加的其他内容。如果您已成功添加任何自定义内容，或者我可能忘记了列表中的任何内容，也可以随时将自定义内容添加到列表中。

接下来是唯一 UUID，为此我们首先要单击生成唯一 UUID，然后单击更新。

VisualResource，如果您还没有，可以在此处生成一个用于合并文件的 ID。但如果您已经设置了合并文件，请复制我们用于 ID 行的 UUID，然后单击更新。

我们现在已设置好文件，因此接下来，如果我们没有现有的 CharacterCreationAppearanceVisuals 文件，请使用另存为按钮。导航到 ModName/Public/YourShared/CharacterCreation 文件夹并保存您的文件。

如果您已有文件，请使用定位您的 CharacterCreationAppearanceVisuals.lsx 按钮并导航到您的文件。设置好后，底部框将更改为您的路径。现在，如果您想向现有文件添加新插槽，可以单击添加到您的模组文件。

恭喜，您已使用迷你工具创建了 CharacterCreationAppearanceVisuals。

#### 将模组种族添加到迷你工具 CCAV 选项

如果想使用迷你工具的单插槽创建功能，可以将自定义种族添加到工具附带的 LSX Files 文件夹中的 Races.txt。它使用此文件夹中的文件作为工具。因此，您可能需要在编辑前备份此文件夹，以防出现任何错误。

打开 Races.txt 后，您将看到以下内容：

RaceUUID，因此您可以快速将上述内容替换为您想要的种族

模组种族

要将新种族添加到列表中，请按上述方式添加它们。种族名称后跟冒号和 RaceUUID。

顶部的行您可能认出是 CharacterCreationAppearanceVisuals 模板底部的 UUID。这是因为我的设置方式是它首先读取名称，冒号后的任何内容都应该是种族 UUID。

因此，要添加自定义种族，您可以将它们添加在模组种族下，也可以将它们添加到顶部列表中，我只是为了方便使用而将它们分开。

因此，您需要遵循相同的格式，键入您希望在下拉菜单中显示的名称，后跟冒号和空格，然后是您/该自定义种族的 UUID。

#### 迷你工具 32 个 CCAV 条目一次性选项

如果您正在为所有种族制作头发，并且更喜欢使用 CCAV 文件，可以使用 32 个 CCAV 条目一次性选项将它们一次性添加到需要添加到使用头发的原版种族的 32 个条目中。因此除了龙裔之外的所有种族。

打开后如下所示：

首先，您需要单击生成唯一句柄或复制您在本地化文件中创建的句柄，然后单击更新按钮。

完成一个部分后始终按更新。

您会注意到这缺少 CharacterCreationAppearanceVisuals 部分中的某些内容。这是因为它已经为所有种族设置好了。因此我们这里不需要下拉菜单。

在名称部分添加您的资源的名称，这将在您的文件中添加注释，以便在需要时更好地找到它。填写资源名称后按更新。

插槽名称的工作方式与其他文件相同。

您可能需要单击两次才能显示整个列表。对于我们的需求，我们希望选择头发。然后单击更新。但列表的其余部分是可以通过 CCAV 添加的其他内容。如果您已成功添加任何自定义内容，或者我可能忘记了列表中的任何内容，也可以随时将自定义内容添加到列表中。

VisualResource，在您的资源合并中复制我们用于 ID 行的 UUID，然后单击更新。

接下来单击生成唯一 UUID，这将把每个唯一 UUID 行更改为每个插槽的唯一 UUID。没有重复。

我们现在已设置好文件，因此接下来，如果我们没有现有的 CharacterCreationAppearanceVisuals 文件，请使用另存为按钮。导航到 ModName/Public/YourShared/CharacterCreation 文件夹并保存您的文件。

如果您已有文件，请使用定位您的 CharacterCreationAppearanceVisuals.lsx 按钮并导航到您的文件。设置好后，底部框将更改为您的路径。现在，如果您想向现有文件添加新插槽，可以单击添加到您的模组文件。

恭喜，您已使用迷你工具为所有 32 个原版插槽创建了 CharacterCreationAppearanceVisuals。

#### 将模组种族添加到迷你工具 32 个 CCAV 条目一次性选项

如果要将自定义/模组种族添加到 32 插槽选项，这是可能的，您只需转到工具附带的 LSX Files 文件夹（确保备份文件夹以防万一）并编辑 CharacterCreationAppearanceVisuals_ForAll.lsx。

在这里，您将找到如下注释以及特定种族的注释。我这样做是为了文件的组织。它在游戏中没有任何作用。

复制任何条目，例如：

<node id="CharacterCreationAppearanceVisual">
<attribute id="BodyShape" type="uint8" value="0" />
<attribute id="BodyType" type="uint8" value="0" />
<attribute id="DisplayName" type="TranslatedString" handle="h70718c5dg76b9g402cgbe76ge5b2d9abaa1f" version="1" />
<attribute id="RaceUUID" type="guid" value="b6dccbed-30f3-424b-a181-c4540cf38197" />
<attribute id="SlotName" type="FixedString" value="Hair" />
<attribute id="UUID" type="guid" value="259cf9c6-c6f1-4265-b4b8-dd45f2a3772f" />
<attribute id="VisualResource" type="guid" value="00000000-0000-0000-0000-000000000000" />
</node>

复制后，我们将 RaceUUID 行更改为自定义/模组种族的 UUID。

<attribute id="BodyType" type="uint8" value="1" />

再次复制整个片段，然后为女性形态（如果该种族有女性形态，因为当前设置为男性），将 Bodytype 从 0 更改为 1（女性）。

<attribute id="BodyShape" type="uint8" value="1" />

如果该种族有强壮体型，再次复制并将 bodyshape 值从 0 更改为 1。如果默认体型是强壮体型，则将其保留为 0。

这就是需要更改的全部内容。我的代码中不需要更改任何内容，因此您无需担心，根据需要复制这些部分。然后像往常一样使用该工具，它将编辑 32 个原版条目以及您添加的所有模组条目。
